# Lv10以降のやりがい設計書 v1（フェーズ6）

## 1. 目的と設計原則

最終進化（完全体・Lv10）到達後の継続動機を設計する。

- **最上位目的**: 子どもの食事モチベーション維持。「明日の食事でもタップしたい理由」を作り続けることが全設計の判断基準。ゲーム的報酬はその手段。
- **不変の制約**:
  - 進化キャラクターの数（56種）は変更しない
  - 進化タイミング（Lv3 / Lv6 / Lv10）は変更しない
  - Lv1〜10のレベル曲線・ポイント計算は変更しない

## 2. 全体構造：二本立て

Lv10以降は2つのルートが共存し、どちらを選んでも「次の食事の理由」がある。

| ルート | 内容 | 動機 |
| --- | --- | --- |
| 継続ルート（デフォルト） | 同一個体が無制限に巨大化 | 「もっと大きくしたい」 |
| 卒業ルート（推奨誘導） | 殿堂入り→新しい赤ちゃんから再スタート | 「次はどの子に会えるかな」「図鑑を埋めたい」 |

## 3. 継続ルート：無限巨大化

### 3-1. サイズモデル
- Lv10以降、レベルアップごとにモンスターのサイズが無制限に増加する。
- サイズ（メートル）: `size(level) = 2.0 × 1.2^(level - 10)`（Lv10=2m、Lv15≈5m、Lv20≈12m、Lv30≈77m、Lv50≈2.9km）
- 表示は子どもが読める単位で整形する（`5m` → `2.9km` など）。
- レベル曲線は既存のまま（必要pt = `(3+2L)×10` の線形増加）。1食あたりの獲得ptはほぼ一定のため、レベルアップ頻度はLv10台=毎食数回 → Lv50頃=1食1回 → Lv100頃=数食に1回と自然減速し、演出の消費速度も自然に落ちる。

### 3-2. はみ出し表現（常時）
- ホーム＝タップ画面は**常に**現在サイズの縮尺で描画する。CSS transform の scale でモンスター画像を拡大し、ステージ領域からはみ出させる（stage は overflow hidden）。
- 描画スケール: `cssScale(level) = min(SCALE_CAP, 1.15^(level - 10))`。レベルが上がるほど「全身→顔→目→鱗一枚」と映る部位が狭くなることで巨大さを暗示する。
- タップ反応（eat/idle/evolve アニメ）は映っている部位＋画面シェイクで表現する。高レベルほどシェイクを強くしてよい。
- **ズームアウト操作は提供しない**。全身は §3-3 のリザルト画面と図鑑でのみ見られる（希少性の維持）。

### 3-3. リザルト画面（新設）
- タップ画面に「ごちそうさま」ボタンを新設し、押すと食事セッション終了→リザルト画面を表示する。
- リザルト画面の内容:
  - **全身遠景カット**: カメラが引いた全身表示（scale=1相当）。レベルが上がるほど演出上「遠くから撮っている」見せ方にする（背景の縮小・余白拡大など）。完食の報酬として機能させる。
  - 今回の記録: たべたかいすう / かくとくpt / さいだいコンボ / 現在レベル / 現在サイズ
  - 卒業バナー（§4-2 の条件を満たすときのみ）

## 4. 卒業ルート：世代交代と殿堂

### 4-1. 卒業（殿堂入り）
- 卒業可能条件: 完全体（evolutionStage=3）かつ Lv10以上。
- 卒業すると現在の個体は**殿堂（Hall of Fame）**に永久保存され、新しい赤ちゃん（Lv1・まっさら）が始まる。
- 殿堂レコード: モンスター種別・表示名・最終レベル・最終サイズ・一緒に食べた回数（生涯累計タップ数）・卒業日。
- 殿堂はメニューからいつでも閲覧できる。**引き継ぎ要素は一切なし**（図鑑・殿堂の記録のみが残る）。周回ボーナス・コスメ継承は行わない（Lv1〜10のペーシング保護と殿堂の唯一無二性のため）。
- 既存メニューの「新しいモンスターにする」（確認ダイアログ＋記録消去）は本フローに**置き換える**。

### 4-2. 卒業推奨の誘導
- **Lv10到達時（初回のみ）**: 最終進化の祝福演出に続けて、卒業選択画面を一度だけフル表示する（「そつぎょうして でんどうに かざる？ / もっと おおきく そだてる！」の対等な二択）。
- **以降Lv15, 20, 25, …（5レベルごと）**: リザルト画面に「そつぎょうできるよ」バナーを控えめに表示するのみ。モーダルは出さない（ナグ化・反射タップ事故の防止）。
- 誘導表示済みレベルを状態に記録し、同一レベル帯で重複表示しない。

### 4-3. 誤発動防止と復元
- 卒業は二段確認: 選択 → 確認画面で**長押し3秒**（「ながおしで おわかれの あいさつをしよう」として儀式的に意味づける）。
- 卒業実行時、直前の状態スナップショットを localStorage（`mgu_graduation_backup_v1`）に日付つきで1件保存する。
- **当日中に限り**、保護者向けメニューから卒業を取り消して復元できる。翌日以降はバックアップ無効（新周回の進行とのデータ矛盾防止）。復元時は殿堂レコードも巻き戻す。

## 5. 図鑑と食育の整合

- 未入手モンスターは**シルエット表示**（画像を黒塗り・名前は「？？？」）。
- 子ども向けメッセージは個別攻略ヒントではなく多様性框架に統一する:
  「いろんな ごはんを たべていると、いろんな子に あえるよ！」
- **保護者向け進化ヒント**をメニュー内に新設する。文面は漢字・英語を多めにし、子どもには意図的に読みにくくする。例:
  「【保護者向け / For Parents】進化分岐は各段階（Lv1-3, Lv3-6）で最も多く選択された食材の属性で決定されます。全食材が15%以上の場合は Balance Route に分岐します。多様な食材選択が図鑑コンプリートの最短経路です。」

## 6. 状態・データ設計（実装指針）

MonsterEngine の状態に以下を追加（`normalizeMonsterState` で後方互換を維持）:

```
lifetimeEatCount: number        // この個体の生涯累計有効タップ数
hallOfFame: [{                  // 卒業した個体の記録（永続・世代を超えて保持）
  monsterKey, displayName, finalLevel, sizeMeters, eatCount, graduatedAt
}]
lastGraduationPromptLevel: number  // 誘導表示済みレベル（重複表示防止）
```

追加API（例）:
- `sizeForLevel(level)` / `formatSize(meters)` / `cssScaleForLevel(level)`
- `canGraduate(state)`
- `shouldShowGraduationPrompt(state)`（Lv10初回 or 5レベルごと判定）
- `graduate(state, now)` → 殿堂レコード追加＋obtainedMonsters/hallOfFame を引き継いだ初期状態を返す

## 7. KPIとの接続

- リザルト画面の新設は「セッション終了」を初めてUI上のイベントにするため、KPI-02（セッション完了率）の計測点となる。
- 卒業→周回は KPI-05（進化到達率）の継続的な再発生を生む。
- 図鑑の多様性メッセージ＋バランスルート判定により、図鑑コンプ動機と食の多様性が同方向を向く。

## 8. 本設計で決定済みの選択（経緯）

| # | 論点 | 決定 |
| --- | --- | --- |
| 1 | やりがいの定義 | 子どもの食事モチベーション維持が主 |
| 2 | 基本構造 | 同一個体の継続育成（巨大化）＋卒業周回の二本立て |
| 3 | 巨大化の表現 | はみ出し方式（面白さ優先） |
| 4 | はみ出しの適用 | 常時（ホーム画面が常にその縮尺） |
| 5 | 全身を見る手段 | 食事終了リザルト画面のみ（＋図鑑） |
| 6 | 卒業後の扱い | 殿堂入り（記録永久保存） |
| 7 | 誘導の強さ | Lv10初回のみ儀式、以降5レベルごとにリザルトバナー |
| 8 | 誤発動防止 | 二段確認（長押し）＋当日中は保護者メニューから復元可 |
| 9 | 図鑑の見せ方 | シルエット＋多様性メッセージ、保護者向けに正確なヒント（漢字・英語多め） |
| 10 | 周回引き継ぎ | なし |
