# 基本設計書 v1（フェーズ1）

## 1. システム概要
MoguMoguMonsterは、子どもの「一口食べる→タップする」行動を入力として、
モンスター育成体験を提供するWebサービスである。フェーズ1ではMVP実装に必要な
画面、ドメイン、API、データ構造の骨格を定義する。

## 2. アーキテクチャ（MVP）
- フロントエンド: スマートフォン向けWeb UI
- バックエンド: REST API
- DB: セッション・イベント・成長状態を保持するRDB
- 分析: イベントログを蓄積しKPI計測に利用

## 3. 主要ドメイン
- User（保護者管理アカウント）
- ChildProfile（子どもプロフィール）
- MealSession（食事セッション）
- TapEvent（タップイベント）
- ComboState（コンボ状態）
- MonsterState（モンスター成長状態）
- FoodAttributeLedger（エサ属性累積）

## 4. MVP境界
### In Scope
- セッション開始/終了
- 有効・無効タップ判定
- コンボ計算
- 成長値加算とレベルアップ
- エサ属性入力と進化分岐
- 結果表示と履歴表示

### Out of Scope
- カメラ自動判定
- 音声自動判定
- リアルタイム対戦/協力機能

## 5. 非機能方針（設計入力）
- 応答速度: 体感遅延を避ける
- 可用性: セッション途切れ時の再開容易性
- セキュリティ: 認証・認可・暗号化
- 観測性: KPIイベントを必須記録
