# 要件定義書 v1（フェーズ0）

## 1. 機能要件
### FR-01 セッション管理
- 食事セッションを開始/終了できる。
- セッション単位でポイントと履歴を保存する。

### FR-02 タップ入力
- 子どもが一口食べた後にタップできる。
- 最短間隔未満の連打は無効扱いにできる。

### FR-03 コンボ判定
- 規定時間内の有効タップでコンボ継続。
- コンボ段階に応じて獲得ポイント倍率を適用。

### FR-04 成長システム
- 累積ポイントでモンスターが段階的に成長。
- 成長閾値を管理画面/設定で調整可能にする（将来拡張前提）。

### FR-05 エサ種別
- タップ時にエサ種別を選択できる。
- エサ種別の蓄積比率で進化分岐先を決定する。

### FR-06 結果表示
- セッション終了時にポイント、コンボ、成長進捗を表示。

### FR-07 保護者向け履歴
- 日単位でセッション結果を閲覧できる。

## 2. 非機能要件（初期）
### NFR-01 性能
- 一般的なスマートフォンで操作遅延を体感しにくいレスポンスを維持する。

### NFR-02 可用性
- 障害時でもセッション継続不能が最小になるよう復旧手順を持つ。

### NFR-03 セキュリティ
- 個人情報の最小収集。
- 通信の暗号化。
- 認証・認可を実装する。

### NFR-04 プライバシー
- 子どものセンシティブ情報を過度に保持しない。
- 利用目的を保護者に明示する。

### NFR-05 運用
- 主要KPIを定期的にモニタリング可能にする。

## 3. ユースケース（MVP）
- UC-01: 子どもがセッション開始し、食べるたびにタップしてモンスターを育てる。
- UC-02: 保護者が当日の結果を確認する。

## 4. 受け入れ条件（フェーズ0時点）
- MVPのIn/Out Scopeが明文化されていること。
- 機能要件FR-01〜07に対して関係者合意があること。
- NFR方針が明文化され、フェーズ1で設計可能な粒度であること。
