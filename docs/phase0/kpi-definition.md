# KPI定義書（フェーズ0）

## 1. KPI設計方針
- フェーズ0では「継続利用」「食事中行動」「体験品質」の3軸で定義する。
- 数値閾値は初期仮説とし、ソフトローンチで再調整する。

## 2. 主要KPI
### KPI-01 週次継続率（W1 Retention）
- 定義: 初回利用ユーザーのうち、7日以内に再利用した割合
- 目的: 習慣化の初期成立を測る

### KPI-02 セッション完了率
- 定義: 開始した食事セッションのうち、終了処理まで到達した割合
- 目的: 体験の離脱しにくさを測る

### KPI-03 有効タップ率
- 定義: 全タップ中、有効判定となったタップの割合
- 目的: 空連打抑制と遊び方の理解度を測る

### KPI-04 平均コンボ長
- 定義: 1セッション内で成立したコンボの平均継続数
- 目的: テンポよく食べる行動の代理指標

### KPI-05 進化到達率
- 定義: 観測期間内に初回進化に到達したユーザー割合
- 目的: 成長体験の到達可能性を測る

## 3. ガードレール指標
### G-01 異常連打率
- 定義: 最短間隔未満タップの比率
- 目的: 不正/誤操作の兆候監視

### G-02 保護者離脱率
- 定義: 初回オンボーディング中の離脱割合
- 目的: 導入負荷の把握

## 4. 計測イベント（初期）
- session_started
- tap_recorded（valid/invalidフラグ含む）
- combo_updated
- session_finished
- growth_level_up
- evolution_decided

## 5. フェーズ0完了条件
- KPI/ガードレールの定義が合意されている
- 計測イベント一覧がフェーズ1の設計入力として利用可能である
